Upload Missing Content


Before uploading

Fix DATs are available to download here. Please check the fix DATs before uploading. Any uploaded content not listed in the fix DATs will be deleted with the exception of content that will be included in upcoming DAT versions or new DAT files. Fix DATs are updated to the latest versions on a weekly basis.


A list of files which have been uploaded and are pending verification and addition to Myrient is available here. This list is updated every 5 minutes.


What file types can be uploaded

The following file format is recommended for the fastest verification and addition to Myrient:

  • TorrentZip
  • ZIP
  • CHD (please do not compress CHD files, otherwise they will not be verified correctly)
  • Raw/uncompressed files

A utility such as TrrntZip is recommended in order to convert existing zip and 7z files to TorrentZip. If you would like to upload zip or 7z raw files, please add them to a zip archive before converting them to TorrentZip.

You may upload a single TorrentZip archive file with files for multiple different fix DATs inside.


The following file formats are not recommended and will result in slower verification and addition to Myrient:

  • 7Z

All other archive and compression file formats should not be uploaded unless they are listed in the fix DATs.

Uploading an archive or a compression file format other than zip or 7z will result in the file not being verified correctly or being deleted.

Frequently Asked Questions

Why was the directory I was trying to access or the file I was trying to download no longer available?

Since the process of verifying and adding uploaded files to Myrient also renames, moves, and removes directories and content as needed according to the latest DAT files, some URLs may break as a result. Any affected directories or content may be available to access or download from new URLs.


What happens if uploads have begun being verified and added but I still have uploads in progress?

Any files which have finished uploading will be verified and added to Myrient and any files which are still being uploaded will be verified and added the next time the process occurs.


If a file with the same hash and size is present in multiple fix DATs, should I upload the file more than once and with different filenames?

Files present in multiple fix DATs (Redump and TOSEC-ISO for example) do not need to be uploaded more than once. Our backend replicates the file in multiple locations as necessary.


Where can I download the DATs Myrient uses to curate the collections?

You can download the DATs here. DATs are updated to the latest versions on a weekly basis at the same time as fix DATs. Please note that a ROM manager such as RomVault is recommended in order to use the DATs effectively.
